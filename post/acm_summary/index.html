数据存储

存储方式：数组和链表，其余散列表，堆栈，图和树等都是数组链表的特殊操作而已，API不同。

  • 队列与栈：既可用数组也可用链表实现，数组实现要注意动态扩容问题，链表实现没有扩容问题，但是需要额外的空间存储节点指针。
  • 图：两种表示—邻接表和邻接矩阵。邻接表就是链表，比较节省空间，但是操作效率不高；邻接矩阵就是二维数组，操作迅速并且可以使用矩阵运算，但是图如果比较稀疏的话，会消耗不少内存。
  • 散列表：通过散列函数把键映射到一个大数组里。存取效率高，但是需解决散列冲突问题。
  • 树：用数组实现就是堆，通过索引来找儿子父亲。用列表实现就是常见的树，这种方法实现的结构灵活，可以用很多变种。

数组：

  • 优点：连续存储，随机访问；
  • 缺点：因为分配空间连续，所以扩容麻烦；数组中插入删除复杂度高。

链表：

  • 优点：元素不连续，不存在扩容问题，插入删除复杂度低。
  • 缺点：只能顺序访问，而且消耗空间存储节点指针。

数据的基本操作：

数据的操作只有几个：遍历+访问，遍历+访问就是为了增删改查。

那么多种数据结构就是为了在不同的场景下，尽可能搞笑的增删改查。

遍历和访问就两种形式：线性的和非线性的。线性就是 for/while 迭代为代表，非线性就是递归为代表。

算法本质：

算法的本质就是穷举。why？计算机设计出来就是用来供人们计算的，计算机的特点就是快，你算一圈，人家算几万圈。所以计算机解决问题的思路就是大道至简—穷举。

这跟数学的思维不一样的，数学题一般自信官产，找关系，列方程。如果说需要进行大量穷举来寻找答案，那这题八成是写错了。

而计算机解决问题的思维恰恰相反，人类你们去总结公式吧，总结出来更好，总结不出来我们就穷举吧。

不要觉得穷举这件事很简单，穷举关键点：无遗漏，无冗余。

遗漏会导致答案错误，冗余会拖慢算法运行效率。

前者对应如何穷举的问题，后者对应如何聪明的穷举。

如何穷举一般对应暴力解法，通过暴力解决问题。

如何聪明的穷举一般对应动态规划，查并集，贪心算法。

二叉树系列：

二叉树递归解法的两种思路：

  • 遍历二叉树得出答案——回溯算法
  • 通过分解问题计算出答案——动态规划

二叉树解法注意——前中后序遍历的不同效果。

//遍历
// 记录最大深度
int res = 0;
int depth = 0;

// 主函数
int maxDepth(TreeNode root) {
	traverse(root);
	return res;
}

// 二叉树遍历框架
void traverse(TreeNode root) {
	if (root == null) {
		// 到达叶子节点
		res = Math.max(res, depth);
		return;
	}
	// 前序遍历位置
	depth++;
	traverse(root.left);
  depth--;
  depth++;
	traverse(root.right);
	// 后序遍历位置
	depth--;
}


// 分解问题
// 定义：输入根节点，返回这棵二叉树的最大深度
class Solution {
public:
    int res = 0;
    int maxDepth(TreeNode* root) {
        if(root == nullptr){
            return 0;
        }
      	// 这里不用搞的那么抽象
      	// 只用想：函数返回值就是我所定义的东西
      	// 然后拿到函数返回值操作，操作出你想要的东西，然后返回即可。
        int left = maxDepth(root->left);
        int right = maxDepth(root->right);
        res = max(left,right) + 1;
        return res;
    }
};
