动态规划解题套路

问题概述：

动态规划问题的一般形式就是求最值。

求最值最好的方法就是穷举。

但是题目千变万化，穷举所有解不是个容易的事。唯有列出状态转移方程，才能穷举完成。只要列出状态转移方程，就意味着按照状态转移方程，闭着眼睛也能完成穷举（base case + 方程）。

但是穷举复杂度太高了呀，这时候我们就需要动态规划了。使用动态规划需要判断是否具有最优子结构和重叠子问题。有了这两个性质，我们就可以使用备忘录或者DP表来优化穷举过程。

最优子问题：最优子问题是一类问题的性质，并不是动态规划所专有的。具有这种性质的问题，可以从子问题的最优结果推出更大规模问题的最优结果。

举个很容易理解的例子：假设你们学校有 10 个班，你已经计算出了每个班的最高考试成绩。那么现在我要求你计算全校最高的成绩，你会不会算？当然会，而且你不用重新遍历全校学生的分数进行比较，而是只要在这 10 个最高成绩中取最大的就是全校的最高成绩。

如果一个问题没有最优子结构，那我们就改造问题。

再举个例子：假设你们学校有 10 个班，你已知每个班的最大分数差（最高分和最低分的差值）。那么现在我让你计算全校学生中的最大分数差，你会不会算？可以想办法算，但是肯定不能通过已知的这 10 个班的最大分数差推到出来。因为这 10 个班的最大分数差不一定就包含全校学生的最大分数差，比如全校的最大分数差可能是 3 班的最高分和 6 班的最低分之差。

改造问题，也就是把问题等价转化：最大分数差，不就等价于最高分数和最低分数的差么，那不就是要求最高和最低分数么，不就是我们讨论的第一个问题么，不就具有最优子结构了么？

重叠子问题：有了最优子结构，我们就能使用穷举来解决问题。只有拥有了重叠子问题，这个问题才能用动态规划来做。

动态规划的问题核心还是在于状态转移矩阵，写出状态转移方程，代表着这个题目你有解法，但是可能不是最优的。

如何写出状态转移方程呢？

明确 base case -> 明确「状态」-> 明确「选择」 -> 定义 dp 数组/函数的含义。

明确状态的重要性：状态转移矩阵就是各个状态之间的变化，只有了然所有的状态，才能写出正确的方程。

选择的重要性：选择也叫动作，动作是各个状态转移的依据。只有知道动作，才能知道大问题与哪些子问题有关。

def dp(状态1....状态n){
  for 选择 in 所有可能的选择：
    # 这个状态就是因为选择而牵扯进来的状态
    result = 求最值(result,dp(状态1....状态n))
  return result
}

dp[0][0][...] = base case
for 状态1 in 所有状态1取值：
		for 状态2 in 所有状态2取值：
  		for ...
    		dp[状态1][状态2][...] = 求最值(选择1，选择2...)

递归算法的时间复杂度怎么计算？就是用子问题个数乘以解决一个子问题需要的时间。

一般简单的方法直接求最坏复杂度$O(2^n)$，再乘上状态转移矩阵的复杂度。

总结：

动态规划做题流程：

  1. 明确base case
  2. 找出状态和动作或找出重叠最优子问题。
  3. 写出状态转移方程——这必须先写出来，这个不写出来，这个题基本没有头绪。
  4. 找出重叠子问题。
  5. 使用备忘录或者dp表。
